## 基础功能

### 发送消息

直接向 Bot 发送消息即可开始对话：

```
你好，请帮我分析一下这段代码
```

Bot 会根据消息内容自动选择处理方式：
- **简单问题**：直接回复
- **复杂任务**：进入分析流程
- **命令**：执行对应操作

### 消息格式

支持的消息类型：

| 类型 | 说明 | 示例 |
|------|------|------|
| 文本 | 普通文字消息 | 任何文字 |
| 代码 | 代码块 | ```python...``` |
| 引用 | 引用上下文 | 回复某条消息 |

## 命令详解

### /help - 帮助

显示所有可用命令和说明。

### /start - 开始新会话

清除当前对话历史，开始新的独立会话。

**使用场景**：
- 切换话题时
- 历史消息过多时
- 需要清除上下文时

### /template - 切换模板

切换 Agent 的能力模板，改变回答风格。

**用法**：
```
/template <template-id>
```

**可用模板**：

| 模板 | 说明 | 适用场景 |
|------|------|----------|
| default | 通用助手 | 日常问答 |
| programmer | 程序员助手 | 代码编写、调试 |
| writer | 作家助手 | 写作、润色 |
| trader | 交易员助手 | 市场分析 |
| researcher | 研究员助手 | 深度分析 |
| pm | 产品经理助手 | 需求分析 |

**示例**：
```
/template programmer

现在我是你的程序员助手，有什么可以帮你的吗？

帮我写一个 React Hook 实现防抖功能
```

### /status - 查看状态

显示当前 Agent 的详细状态信息。

**输出示例**：
```
📊 Agent 状态:
- 名称: 我的助手
- 模板: programmer
- 记忆: 开启
- 模型: kimi-latest
- 会话消息: 24
- 运行时间: 2小时
```

### /memory - 记忆开关

开启或关闭长期记忆功能。

**用法**：
```
/memory on   # 开启
/memory off  # 关闭
```

**说明**：
- 开启后，Bot 会记住用户偏好、重要信息
- 记忆在会话间持久化
- 可以随时清除

### /reset - 重置会话

清除当前会话的所有上下文，重新开始。

**与 /start 的区别**：
- `/reset`：保留长期记忆，清除会话历史
- `/start`：完全新的开始

### /task - 任务管理

查看和管理进行中的任务。

**用法**：
```
/task list    # 列出所有任务
/task status  # 查看任务状态
```

## 角色模板

### 内置模板

#### 1. Default（默认）

通用型助手，适合日常问答。

**特点**：
- 平衡的回答风格
- 广泛的知识覆盖
- 适合多领域问题

#### 2. Programmer（程序员）

专注于软件开发。

**特点**：
- 代码优化建议

- 架构设计
- 最佳实践

**示例对话**：
```
用户：这段代码有性能问题吗？
Bot：[详细分析代码并提供优化建议]
```

#### 3. Writer（作家）

专注于写作和内容创作。

**特点**：
- 文案润色
- 创意写作
- 结构优化

#### 4. Trader（交易员）

专注于金融市场分析。

**特点**：
- 技术分析
- 基本面分析
- 风险管理
- 策略回测

#### 5. Researcher（研究员）

专注于深度研究和分析。

**特点**：
- 文献综述
- 数据分析
- 报告撰写
- 假设验证

#### 6. PM（产品经理）

专注于产品设计和需求分析。

**特点**：

## 上下文管理

### 会话上下文

每个对话会话独立维护上下文：

- **消息历史**：最近的对话记录
- **状态信息**：当前对话状态
- **临时数据**：本次会话的临时信息

### 长期记忆

跨会话持久化的信息：

- **用户画像**：姓名、职业、偏好
- **项目信息**：项目背景、技术栈
- **重要事实**：关键信息和决策

**示例**：
```
用户：我叫李四，是一名 Java 后端工程师
[记忆已保存]

用户：帮我设计一个用户系统
Bot：好的李四，作为 Java 后端工程师，我建议使用 Spring Boot + MyBatis Plus 架构...
```

### 上下文窗口

系统限制：
- 最大消息数：20 条
- 最大字符数：4000

超出限制时，较早的消息会被压缩或移除。

## 任务类型

### Direct（直接执行）

简单任务直接回复。

**示例**：
- 问答
- 简单代码片段
- 概念解释

### LongTask（长任务）

耗时任务后台执行。

**示例**：
- 代码重构
- 批量数据处理
- 深度分析

**特点**：
- 异步执行
- 进度通知
- 后台完成

### FlowTask（流程任务）

需要人机协作的多步骤任务。

**示例**：
- 系统部署
- 数据迁移
- 架构设计

**流程**：
1. Bot 提出方案
2. 用户确认
3. 执行步骤
4. 重复直到完成

## 💡 最佳实践

```

```

根据场景选择合适的角色模板：


定期清理不再需要的记忆：

```
/memory off  # 临时关闭
/memory on   # 重新开启
```

### 4. 分段处理

复杂任务分段处理：

```
我们先设计数据库表结构，然后再写 API 接口
```

## 🆘 常见问题

### Q: Bot 忘记了之前说的话？

A: 检查上下文窗口是否已满，或使用 `/reset` 重新开始。

### Q: 回复速度很慢？

A: 可能是复杂任务正在分析中，或使用流式响应模式。

### Q: 如何清除所有记忆？

A: 删除 `~/.weixin-kimi-bot/memory/` 目录。

### Q: 支持哪些编程语言？

A: 支持所有主流编程语言，包括但不限于 Python、JavaScript、Java、Go、Rust 等。
